# 贡献指南

感谢你帮助 Awesome Zhuiju Free 保持真实、及时和透明。提交内容前，请先阅读以下规则。

## 可以提交什么

- 在线影视站与官方免费频道
- 网盘资源搜索与文件管理工具
- 磁力链接搜索与合法内容分发工具
- 字幕站、字幕组与字幕工具
- TVBox、Kodi 等播放器及其生态工具
- IPTV、广播和其他公开订阅源
- 会员拼团、家庭套餐与省钱信息

项目可以收录存在争议或高风险的资源，但必须如实说明风险。以下内容不接受：

- 单部影视作品的盗版播放、下载或磁力链接
- 影视文件、破解软件、账号、密钥或绕过付费限制的工具
- 恶意软件、钓鱼、诈骗、成人或明显违法内容
- 隐藏推广关系、伪造验证记录或操纵评分的提交

## 推荐新资源

优先使用 GitHub 的“推荐新资源”Issue 模板。提交时至少提供：

1. 资源名称、主页 URL 和分类。
2. 一句话说明它解决什么问题。
3. 是否需要登录、付费或安装软件。
4. 已知广告、跳转、隐私、版权与支付风险。
5. 你的测试日期、地区和网络环境。

维护者确认后，会为资源分配稳定的 `id` 并加入 `resources/resources.json`。

## 评分规则

每个能力维度使用 1.0 至 5.0 分，可以使用一位小数。

| 分数 | 含义 |
| ---: | --- |
| 5 | 同类资源中表现优秀，几乎没有明显问题 |
| 4 | 表现良好，存在少量不影响主要使用的问题 |
| 3 | 基本可用，有明显限制或体验问题 |
| 2 | 勉强可用，问题较多 |
| 1 | 几乎不可用或体验非常差 |

### 能力维度

- **多**：资源数量、类型覆盖、更新速度。
- **快**：页面打开、搜索响应、播放或下载入口加载速度。
- **净**：广告、弹窗、跳转、追踪和诱导下载情况，越高越干净。
- **稳**：长期可访问性、域名稳定性、维护频率。
- **易**：界面、搜索、移动端体验、使用门槛。

### 风险等级

版权、安全、隐私和支付风险分别标记：

- `low`：未发现明显风险，或来源与许可清晰。
- `medium`：存在争议、用户上传内容、较多追踪或其他可控风险。
- `high`：存在明显侵权投诉、恶意行为、账号或支付风险。
- `unknown`：没有足够证据判断。

风险等级不参与能力评分。资源丰富不代表安全或合法。

## 验证规则

一次有效验证至少记录：

- 验证日期和验证者
- 测试地区与网络环境
- 主页是否可访问
- 核心功能是否可使用
- 广告、跳转、诱导下载和登录要求
- 新发现的风险或状态变化

验证记录追加到 `reports/verifications.json`。不要覆盖历史记录。

## 状态与下架

资源状态使用以下值：

- `pending`：待验证
- `verified`：已验证
- `recommended`：推荐
- `caution`：谨慎使用
- `temporarily_unavailable`：暂时失效
- `removed`：已下架

出现以下情况时，资源会降级或下架：

- 连续多次验证无法访问
- 出现恶意软件、钓鱼、诈骗或严重隐私风险
- 资源描述与实际行为严重不符
- 收到有效权利通知，复核后需要断开链接
- 维护者认为继续收录会给用户造成明显伤害

下架不会删除历史。处理原因和时间应记录在 `reports/notices.json` 或验证记录中。

## 权利通知与投诉

权利人请求应尽量包含：

1. 权利人或授权代表的联系方式。
2. 涉及的具体资源 URL。
3. 权利基础与可核验材料。
4. 希望采取的处理措施。
5. 信息真实准确的声明。

维护者收到请求后会先隐藏不必要的个人信息，再进行复核并记录处理结果。紧急安全问题可以在 Issue 中仅提供概况，等待维护者联系。

## Pull Request 检查清单

- JSON 文件可以正常解析，并符合 `resources/schema.json`
- 没有添加单部影视作品链接或受限制内容
- 描述中立，没有营销话术
- 评分有验证依据，风险没有被弱化
- 日期使用 `YYYY-MM-DD`
- 修改资源状态时同步追加验证或投诉记录
